# 俄羅斯方塊 (Tetris) 專案教材導覽 

歡迎來到 Tetris 專案的學習之旅！這份教材旨在幫助初學者透過實作小遊戲，循序漸進地學習 Java 視窗程式設計（Swing）與物件導向觀念。

---

## 1. 專案簡介

這是一套階段式開發的俄羅斯方塊教學專案。我們將龐大的遊戲開發過程拆解為數個容易消化的階段。

**目前專案進度：第二階段「移動、碰撞、消橫列與計分」**
目前的程式碼已經建構出了遊戲的核心可玩性，包含：
* 跳出 Windows 遊戲視窗並畫出 10x20 的標準棋盤格線。
* **自動下落**：引入計時器 (`Timer`) 做為遊戲迴圈的心跳。
* **鍵盤操作**：具備左、右移動，旋轉，以及軟、硬掉落機制。
* **碰撞偵測**：方塊不會出界，也不會與已經固定的方塊重疊。
* **鎖定與消除**：方塊鎖定到底部後，系統會檢查滿排進行消去，並累加總分與行數。
（請注意：目前的版本雖具備完整規則，但尚未加上側邊欄計分板、下一個方塊預覽等完整的 UI，這將保留到最後的第三階段）。

## 2. 文件用途說明

為了幫助你從不同角度理解專案，我們準備了三份文件，各自擁有不同的任務：

* 📍 **`PROJECT_OVERVIEW.md` (本文)**
  * **用途**：整組教材的「入口頁」與「導覽地圖」。
  * **內容**：提供學習方向、建議閱讀順序及原始碼的閱讀路線。

* 🏗️ **`OverviewUML.md` (架構導向)**
  * **用途**：幫助你建立上帝視角，理解程式碼的「空間與結構」。
  * **內容**：專注於解釋有哪些類別、各類別負責什麼工作、類別與類別之間的依賴關係（UML 中的 `has-a` 與 `is-a`），以及物件如何互動。

* 👣 **`progression.md` (演化導向)**
  * **用途**：幫助你理解專案發展的「時間與歷程」。
  * **內容**：說明每個開發階段（目前到第二階段）達成了什麼目標、新增了哪些具體功能，以及玩家情境與系統流程。

## 3. 建議閱讀順序

為了最平滑的學習曲線，建議依循以下順序進行：

1. **先編譯並執行現有程式**（`javac *.java` 後執行 `java Tetris`），看看目前畫面上長什麼樣子。
2. **閱讀 `PROJECT_OVERVIEW.md` (本文)**：掌握整體學習方向。
3. **閱讀 `OverviewUML.md`**：了解目前有哪些類別組成，搞懂它們的分工。
4. **閱讀 `progression.md`**：看懂目前我們完成的功能目標與系統的執行順序。
5. **進入原始碼閱讀**（請參考下方的原始碼閱讀路線）。

---

## 4. 原始碼閱讀路線

當你準備好打開 `.java` 檔案時，建議遵照以下順序閱讀，從小到大、從基礎到核心來建立概念：

1. 🟢 **`Tetris.java`**
   * **關鍵看點**：看一下 `main` 方法如何建立出 `JFrame` 視窗，並把 `TetrisPanel` 塞進去。這裡的程式碼非常短。
2. 🟡 **`TetrominoType.java`**
   * **關鍵看點**：了解 Java 中的 `enum` 列舉是如何被靈活運用的。看看 7 種方塊（I, J, L...）的形狀二維陣列與顏色是如何被定義死在這裡的。
3. 🟡 **`Tetromino.java`**
   * **關鍵看點**：理解這只是一個極度單純的「資料載體」。它裝載了屬於自己的 `TetrominoType`，以及紀錄自己目前的座標（x, y）與旋轉狀態。
4. 🔴 **`TetrisPanel.java`**
   * **關鍵看點**：這是目前專案的核心邏輯區。不要一次全部看完！建議依照順序追蹤：
     * **建構子** (`Timer` 的啟動)、**輸入綁定** (`setupKeyBindings`)。
     * **主要遊戲迴圈** (`gameStep()`)。
     * **核心碰撞演算法** (`isValidPosition()`)：看懂方塊未來座標是怎麼跟牆壁或底部固化方塊 (`board`) 比較。
     * **滿行消去** (`clearCompletedLines()`)。
     * 最後才是 `paintComponent()` 如何把這些動態資料畫成圖。

---

## 5. 適合初學者的學習建議

* **學習尋找進入點 (Entry Point)**：看大型類別 (如 `TetrisPanel`) 時，先從誰「呼叫」了這個功能開始。例如：要了解方塊怎麼下落，就去找 `Timer` 啟動了什麼 (`gameStep`)；要了解方向鍵，就去找 `InputMap` 綁定了誰 (`moveCurrentPiece`)。
* **分清介面繪圖與抽象邏輯**：所有的碰撞偵測 (`isValidPosition`) 與消除橫列 (`clearCompletedLines`) 完全不牽涉 `Graphics` 畫圖元素！它們只跟純數學與陣列 (`Color[][] board`) 有關。當數學算完，最後才統一讓 `repaint()` 生出畫面。
* **動手改改看**：建議試著：
  * 修改 `NORMAL_DROP_DELAY` 或 `FAST_DROP_DELAY` 變數來調整遊戲與按著下的難度節奏。
  * 看看 `linesCleared` 是怎麼累加的，甚至可以試著印出文字 `System.out.println("加分了！")` 觀察。
  * 修改 `clearCompletedLines()` 中的給分常數。
* 當你透徹理解遊戲引擎是如何判定撞牆以及滿行的，我們就能前進第三階段，加入美觀的分數表與預告方塊系統了！
