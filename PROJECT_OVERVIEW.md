# 俄羅斯方塊 (Tetris) 專案教材導覽 

> **🚀 快速啟動 (Quick Start)**
> 如果你想立刻立刻編譯並執行目前的遊戲，請複製以下指令貼入終端機 (Terminal) 中執行：
> ```bash
> javac -encoding UTF-8 *.java
> java Tetris
> ```

歡迎來到 Tetris 專案的學習之旅！這份教材旨在幫助初學者透過實作小遊戲，循序漸進地學習 Java 視窗程式設計（Swing）與物件導向觀念。

---

## 1. 專案簡介

這是一套階段式開發的俄羅斯方塊教學專案。我們將龐大的遊戲開發過程拆解為數個容易消化的階段。

**目前專案進度：第三階段「系統優化與完整遊戲體驗」**
目前的程式碼已經建構出了完整的遊戲體驗，包含：
* **核心可玩性**：自動下落、完整鍵盤操作、精準的陣列碰撞偵測與滿列消除計分。
* **進階 UI 體驗**：加入幫助判定落點的影子透明方塊 (Ghost Piece) 以及下一個方塊預覽看板。
* **狀態與音效**：透過布林狀態機加入遊戲暫停、Game Over，並完美整合了復古的 MIDI 事件電子音效！

為了幫助你從不同角度理解專案，我們準備了三份文件，各自擁有不同的任務：

  * **用途**：整組教材的「入口頁」與「導覽地圖」。
  * **內容**：提供學習方向、建議閱讀順序及原始碼的閱讀路線。

* 🏗️ **`OverviewUML.md` (架構導向)**

## 3. 建議閱讀順序

為了最平滑的學習曲線，建議依循以下順序進行：

1. **直接玩玩看**：複製最上方的編譯執行指令 (`javac` 與 `java`)，親自體驗目前的遊戲系統運作。
2. **閱讀 `PROJECT_OVERVIEW.md` (本文)**：掌握整體學習方向。
3. **閱讀 `OverviewUML.md`**：了解目前有哪些類別組成，搞懂它們的分工。
4. **閱讀 `progression.md`**：看懂目前我們完成的功能目標與系統的執行順序。
5. **進入原始碼閱讀**（請參考下方的原始碼閱讀路線）。

## 4. 原始碼閱讀路線

當你準備好打開 `.java` 檔案時，建議遵照以下順序閱讀，從小到大、從基礎到核心來建立概念：

1. 🟢 **`Tetris.java`**
   * **關鍵看點**：看一下 `main` 方法如何建立出 `JFrame` 視窗，並把 `TetrisPanel` 塞進去。這裡的程式碼非常短。
2. 🟡 **`TetrominoType.java`**
   * **關鍵看點**：了解 Java 中的 `enum` 列舉是如何被靈活運用的。看看 7 種方塊（I, J, L...）的形狀二維陣列與顏色是如何被定義死在這裡的。
3. 🟡 **`Tetromino.java`**
   * **關鍵看點**：理解這只是一個極度的「資料載體」。它裝載了屬於自己的 `TetrominoType`，動態地紀錄自己的座標（x, y）與旋轉狀態。
4. 🔵 **`SoundManager.java` (與音效測試)**
   * **關鍵看點**：純粹負責發聲的輔助類別。看看 Java 是如何利用內建的 MIDI Synthesizer 奏出復古 8-bit 的消除與 Game Over 音效！
5. 🔴 **`TetrisPanel.java`**
   * **關鍵看點**：無庸置疑的專案核心。請依照順序追蹤：
     * **建構子** 裡的 `Timer` 啟動、與 **按鍵輸入綁定** (`setupKeyBindings`)。
     * **遊戲主迴圈與狀態判定** (`gameStep()`)：看懂 `isGameOver` 和 `isPaused` 怎麼阻擋流程。
     * **碰撞演算法** (`isValidPosition()`)：方塊未來座標是如何跟牆壁與滿了的陣列 (`board`) 比較的？
     * **滿行消去與音效觸發** (`clearCompletedLines()`)。
     * **視覺呈現中心** `paintComponent()`：觀察如何依序畫上背景、固化塊、現在方塊、**影子透明方塊** 以及 **右側的 UI 與預告**。

## 5. 適合初學者的學習建議

* **學習尋找進入點 (Entry Point)**：看大型類別時，先找誰「呼叫」了功能。例如：下落是誰負責啟動的 (`Timer.gameStep()`)；方向鍵是誰綁定的 (`InputMap`)；而 Game Over 大字是什麼時候畫出來的？
* **分清介面繪圖與抽象邏輯**：所有的碰撞偵測 (`isValidPosition`) 與消除橫列 (`clearCompletedLines`) 完全不牽涉 `Graphics` 畫圖元素！它們只跟純數學與陣列 (`Color[][] board`) 有關。當數學算完，最後才統一讓 `repaint()` 生出畫面。
* **動手改改看**：請不要怕改壞，建議試著：
  * 修改 `NORMAL_DROP_DELAY` 或 `FAST_DROP_DELAY` 變數改變遊戲掉落節奏。
  * 去 `paintComponent()` 裡找 `AlphaComposite`，試著把影子預判方塊的透明度參數改換！
  * 看看 `SoundManager`，試著把自己喜歡的 MIDI 音符 `note` 編號寫進發生碰撞或是消去的事件裡。
* 當你透徹地玩過並且看懂這三個階段的心血，恭喜你，你已經具備運用 Java 物件導向、陣列以及基礎 2D 結合，去開發完整遊戲的紮實功力了！
