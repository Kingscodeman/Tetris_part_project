# AI 協作與文件重構紀錄 (CODEX_REVIEW_LOG)

這份文件記錄了開發者與 AI (Copilot / Codex) 在本次 `Tetris_part` 專案轉型為「三階段教學教材」的過程與決策。

---

## 紀錄 1：專案備份與退回第一階段

* **問題 / 需求**：
  原專案已開發為完整的俄羅斯方塊遊戲。為了教學需求，希望將開發流程拆分為三個階段（1. 基本類別與棋盤骨架、2. 移動與碰撞消去、3. 系統優化與完整體驗），並請 AI 將當下專案源碼退回到「第一階段」的狀態。
* **討論重點**：
  需要保留一份完整版程式碼作為未來參考樣板，同時簡化目前工作目錄中的 `*.java` 檔案，移除所有與鍵盤操作、方塊掉落（Timer）、碰撞計算、得分相關的邏輯。
* **做出的決策**：
  建立 `Complete_Reference` 資料夾存放完整程式碼。將主要類別 (`TetrisPanel`, `Tetromino`, `TetrominoType`) 閹割至僅具備背景繪圖與「靜態單一測試方塊」繪製的功能。
* **修改了哪些檔案**：
  * `Tetris.java` 
  * `TetrisPanel.java`
  * `Tetromino.java`
  * `TetrominoType.java`
* **最終結果**：
  成功產出第一階段的 Java 骨架。編譯後啟動只會看到深色 10x20 棋盤與極上方的一個不動方塊。

## 紀錄 2：重寫演化歷程教材 (`progression.md`)

* **問題 / 需求**：
  將 `progression.md` 重寫為教學用途，說明專案逐步發展的歷程。
* **討論重點**：
  每個階段的說明必須包含：新增的功能、涉及的主要類別、UML 關係 (has-a / is-a)、使用案例（玩家情境）以及系統流程。文字風格須平易近人。
* **做出的決策**：
  採用文件化 Markdown 格式。先撰寫第一階段的架構歷程（視窗建立、靜態方塊繪製），釐清 `TetrisPanel` (is-a JPanel) 與 `Tetromino` (has-a TetrominoType) 等關係。
* **修改了哪些檔案**：
  * `progression.md`（新建/重寫）
* **最終結果**：
  產出條理分明的歷程說明文檔，以第一階段做為重頭戲進行實作講解。

## 紀錄 3：重構 UML 教學文件 (`OverviewUML.md`)

* **問題 / 需求**：
  將舊有的 `UML.md` 刪除，改寫並更名為 `OverviewUML.md`，內容須專注於「架構導向」的教學。
* **討論重點**：
  這份文件須與 `progression.md` 分工，避免重複內容。要更深入且廣泛地探討每個類別的責任分工、UML 的依賴本質，以及物件互動的主要流程（模擬 Sequence Diagram）。
* **做出的決策**：
  將 `OverviewUML.md` 切分為三大部分：類別責任分工、is-a/has-a 概念解析、主要互動流程；同時移除過時的 `UML.md`。
* **修改了哪些檔案**：
  * 刪除 `UML.md`
  * 新增 `OverviewUML.md`
* **最終結果**：
  學生現在有一份獨立的文件可以隨時查閱整個專案的靜態類別結構圖與互動藍圖。

## 紀錄 5：建立教材入口頁 (`PROJECT_OVERVIEW.md`)

* **問題 / 需求**：
  需要一個統整所有教材的「入口首頁」，且僅限於目前第一階段的認知範圍。
* **討論重點**：
  如何引導完全零基礎或剛拿到專案的學生？給予清晰的文件閱讀先後順序與原始碼追蹤路線是關鍵。
* **做出的決策**：
  規劃出 5 個小節：專案介紹、3 份 md 文件的職責、閱讀文件順序、原始碼追蹤順序（從 `Tetris.java` 到 `TetrisPanel.java`），以及針對初學者的鼓勵（建議動手更改參數）。
* **修改了哪些檔案**：
  * 新增 `PROJECT_OVERVIEW.md`
* **最終結果**：
  整組以「三個階段循序漸進」為核心概念的俄羅斯方塊教材正式成形。第一階段具備了由 `PROJECT_OVERVIEW.md` 領頭的完善靜態教材與程式碼骨架。

## 紀錄 6：進階至第二階段「移動、碰撞、消橫列」實作與除錯

* **問題 / 需求**：
  將目前僅有畫面的第一階段程式碼，推進至可遊玩的第二階段，具備方塊下落、鍵盤操作、碰撞偵測與消除機制。
* **討論重點**：
  如何不依賴額外的 UI 類別，僅在 `TetrisPanel.java` 內透過 `Timer` 實作遊戲迴圈，並加入 2D 陣列來記憶固定方塊（`board`）。期間遭遇了建立 Java 檔案時編碼導致 `javac` 編譯出錯的問題。
* **做出的決策**：
  完善 `TetrisPanel`：加入 `InputMap`/`ActionMap` 處理按鍵綁定，實作 `isValidPosition` 進行碰撞邊界判定。產生 `.java` 檔案時，強調使用 UTF-8 編碼確保含有中文註解的 Java 源碼能夠正確被編譯。
* **修改了哪些檔案**：
  * `TetrisPanel.java`
* **最終結果**：
  順利編譯執行，遊戲具備了完整的方塊掉落、旋轉、貼齊與消除的核心邏輯。

## 紀錄 9：音效系統整合 (SoundManager) 與編碼修正

* **問題 / 需求**：
  為遊戲加入聽覺回饋，涵蓋操作音效（移動、旋轉、硬降、消行、Game Over）及背景音樂，提昇沉浸感。同時需徹底解決之前遇到的 `.java` 檔案編碼問題。
* **討論重點**：
  將還原出的測試檔 `SoundTest.class` / `SoundTest.java` 轉化並實裝進遊戲專案中。需要確保「背景音樂」與「操作音效」分離，兩者不能互相覆蓋或產生聽覺干擾。
* **做出的決策**：
  建立專責處理聲音的 `SoundManager.java`。設定它可以在發動不同事件時被呼叫，並將所有檔案統一強制修正編碼保存。
* **修改了哪些檔案**：
  * `SoundTest.java` (還原與測試)
  * `SoundManager.java` (新建並套用)
  * `TetrisPanel.java` (放入觸發音效的呼叫邏輯)
* **最終結果**：
  音效與背景音樂順利發聲，可透過標準指令 `javac Tetris.java TetrisPanel.java SoundManager.java TetrominoType.java Tetromino.java` 順利編譯，且 `java Tetris` 執行順暢。
